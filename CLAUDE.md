# AI Wiki 治理文件

## 核心理念

用户只做三件事：**存、问、研究**。中间的整理、调研、分析，全是 AI 的事。

## 目录结构

```
├── inbox/              # 待处理（文章入口）
├── raw/                # 原文归档（按月 YYYY-MM/，不可变）
├── context/            # 个人上下文（用户维护，AI 只读）
│   ├── finance.md      #   投资相关：持仓、风险偏好、关注领域
│   └── ...             #   可按需扩展其他上下文文件
├── decisions/          # 决策日志（AI 写入，用户回看）
│   └── YYYY-MM-DD-简短标识.md
├── todos/              # AI 待办管理
│   ├── active.md       #   当前活跃待办（工作/个人分区）
│   └── archive/        #   按周归档（YYYY-WXX.md）
└── wiki/               # AI 维护的知识库
    ├── index.md        # 总索引（AI 入口点，每次操作先读此文件）
    ├── log.md          # 处理日志（只追加）
    ├── sources/        # 文章摘要卡片（一对一，每篇存入的文章一张）
    ├── topics/         # 跨文章聚合主题（同类文章多了自动生成）
    └── analysis/       # 课题研究工作区
        └── {课题名}/
            ├── plan.md       # 研究计划（问题定义、已知/未知、搜索方向）
            ├── materials/    # 搜集的原始资料（网页抓取等）
            ├── notes.md      # 研究笔记（中间产出、信息缺口记录）
            ├── report.md     # 最终研究报告（结论、来源、未解决问题）
            └── follow-ups/   # 持续跟踪项（研究产出中需要跟进的部分）
```

## 文章处理流程

1. 新文章进入 `inbox/`（来源：微信转发、浏览器插件、手动放入）
2. AI 执行 `/digest` 处理：
   - 读取文章全文
   - 生成知识卡片到 `wiki/sources/`
   - 原文移到 `raw/YYYY-MM/`
   - 更新 `wiki/index.md` 索引
   - 追加日志到 `wiki/log.md`
   - 检查是否需要更新已有 topic 或创建新 topic

## 知识卡片格式（sources/）

```markdown
---
title: 文章标题
date: YYYY-MM-DD
tags: [标签1, 标签2]
source: article | self
origin: 原文URL（如有）
related: [[关联页面1]], [[关联页面2]]
---

## 一句话
50字以内的摘要。

## 关键要点
- 要点1
- 要点2
- 要点3

## 原文
→ [[raw/文件名]]
```

`source: self` 的文章（用户自己写的）：保留原文，不重写摘要，只做分类打标签和建立关联。

## 主题页面格式（topics/）

当同类标签的文章积累到 3 篇以上时，自动生成或更新主题页面：

```markdown
---
title: 主题名称
tags: [标签]
updated: YYYY-MM-DD
related: [[相关文章1]], [[相关文章2]]
---

## 概述
基于 N 篇文章的聚合总结。

## 核心发现
- ...

## 关联文章
- [[文章1]] - 一句话摘要
- [[文章2]] - 一句话摘要
```

## 课题研究流程

用户说"研究 XX"时触发。Research 产出分两类：

- **知识型**：产出通用方法论、原理等长期有价值的知识 → 入库到 sources/
- **决策型**：产出时效性分析、个人决策建议等 → 保留在 analysis/，不入库

### 流程

1. **开题**：在 `wiki/analysis/{课题名}/` 下创建 `plan.md`
   - 定义研究问题
   - 读取 `context/` 下相关的个人上下文
   - 扫描知识库已有内容，标记已知/未知
   - 列出搜索方向
   - 向用户确认研究方向，用户可补充/调整

2. **搜集**：搜索 + 抓取
   - web search 搜索相关资料
   - 用 webReader 抓取有价值的内容到 `materials/`
   - 从知识库中挑出已有的相关 sources/topics

3. **研究**：读材料 → 写笔记（可多轮循环 ②③）
   - 逐篇阅读 materials/
   - 在 `notes.md` 记录关键发现、交叉验证、信息缺口
   - 发现缺口时继续搜索补充
   - 可主动向用户汇报阶段性发现，听取反馈调整方向

4. **输出**：写 `report.md`
   - 综合结论
   - 标注信息来源（链接到 materials/ 或外部 URL）
   - 列出未解决问题

5. **讨论与补充**：报告输出后，用户继续讨论追问
   - 将有价值的讨论内容追加到 `notes.md` 的 `## 讨论记录` 部分
   - 按日期组织，格式：`### YYYY-MM-DD：主题`，记录核心观点和结论
   - 讨论产生新决策时，更新 `decisions/`
   - 发现新信息缺口时，更新 notes.md

6. **入库（可选）**：AI 判断产出类型
   - **知识型**（方法论、原理等长期知识）：关键发现 → 新建 `wiki/sources/` 知识卡片，更新 topics/、index.md、log.md
   - **决策型**（时效性分析、个人建议）：报告保留在 analysis/，同时在 `decisions/` 创建决策记录
   - 不确定时，向用户建议并确认："这是时效性分析，建议保留在 analysis/ 不入库，可以吗？"

7. **跟进（强制）**：研究完成后，AI 必须向用户确认跟进项
   - 从报告中找出需要持续跟踪的发现（如数据变化、政策动态、业务指标等）
   - 向用户确认："以下发现可能需要持续跟进：1. xxx 2. xxx，需要创建跟踪项吗？"
   - 用户确认后，在 `follow-ups/` 下创建跟踪文件
   - 同时在 `todos/active.md` 对应分类的「### 长期」下添加跟进待办

### plan.md 格式

```markdown
---
topic: 课题名称
created: YYYY-MM-DD
status: active | completed
---

## 研究问题
要回答的核心问题。

## 已有认知
来自知识库的已有内容（引用 sources/）。

## 信息缺口
还不知道、需要搜索的方向。

## 进度
- [x] 已完成的步骤
- [ ] 下一步
```

## 个人上下文（context/）

`context/` 存放用户的个人资料，供 AI 在研究和问答时作为决策参考。与知识库（关于世界的知识）不同，context 是**关于用户自身的信息**。

- **维护者**：用户自行维护，AI 不修改内容
- **读取时机**：研究或问答涉及个人决策时，AI 读取相关上下文文件
- **识别规则**：AI 根据研究主题自动判断需要读取哪些 context 文件（如涉及投资决策则读 `finance.md`）
- **过期提醒**：AI 发现上下文信息可能过时时，可提醒用户更新

## 决策记录（decisions/）

记录用户的重要决策及后续复盘，用于定期回顾和反思。

### 决策来源

- **研究产出**：决策型研究完成后，自动在 `decisions/` 创建记录
- **用户口述**：用户说"记录一个决策"时手动创建

### 决策记录格式

文件名：`YYYY-MM-DD-简短标识.md`

```markdown
---
title: 决策标题
date: YYYY-MM-DD
tags: [投资/职业/生活]
status: open
review_date: YYYY-MM-DD
context: analysis/xxx   # 关联的研究（如有）
---

## 决策
做了什么决定。

## 当时理由
为什么这样做。

## 预期
期待什么结果。

## 复盘
（待复盘）
```

### 状态流转

- `open` → 待复盘（review_date 到期后由复盘流程处理）
- `reviewed` → 已复盘
- `closed` → 决策已结束，不再跟进

### 复盘流程

由 `/wiki-review` skill 执行，可手动触发或由系统定时任务触发。扫描到期的 open 状态决策，逐条与用户回顾实际结果，记录教训。

## 问答规则

用户提问时：
1. 先读 `wiki/index.md` 了解知识库全貌
2. 根据问题定位相关的 sources/topics/analysis 文件
3. 基于知识库内容回答，标注引用来源
4. 如果知识库中没有相关内容，如实告知

## 标签体系

AI 自动打标签，不预定义。常见维度：
- 领域：技术、股票、生活、健康、读书、工作
- 具体主题：Rust、新能源、量化交易
- 类型：教程、观点、研报、经验

## 必须使用中文

所有生成的文件内容使用中文，文件名可包含英文。
