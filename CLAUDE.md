# AI Wiki 治理文件

## 核心理念

用户只做两件事：**存和问**。中间的整理、分类、摘要，全是 AI 的事。

## 目录结构

```
├── inbox/              # 待处理（文章入口）
├── raw/                # 原文归档（按月 YYYY-MM/，不可变）
└── wiki/               # AI 维护的知识库
    ├── index.md        # 总索引（AI 入口点，每次操作先读此文件）
    ├── log.md          # 处理日志（只追加）
    ├── sources/        # 文章摘要卡片（一对一，每篇存入的文章一张）
    ├── topics/         # 跨文章聚合主题（同类文章多了自动生成）
    └── analysis/       # 主动研究的输出（用户提问时才生成）
```

## 文章处理流程

1. 新文章进入 `inbox/`（来源：微信转发、浏览器插件、手动放入）
2. AI 执行 `/digest` 处理：
   - 读取文章全文
   - 生成知识卡片到 `wiki/sources/`
   - 原文移到 `raw/YYYY-MM/`
   - 更新 `wiki/index.md` 索引
   - 追加日志到 `wiki/log.md`
   - 检查是否需要更新已有 topic 或创建新 topic

## 知识卡片格式（sources/）

```markdown
---
title: 文章标题
date: YYYY-MM-DD
tags: [标签1, 标签2]
source: article | self
origin: 原文URL（如有）
related: [[关联页面1]], [[关联页面2]]
---

## 一句话
50字以内的摘要。

## 关键要点
- 要点1
- 要点2
- 要点3

## 原文
→ [[raw/文件名]]
```

`source: self` 的文章（用户自己写的）：保留原文，不重写摘要，只做分类打标签和建立关联。

## 主题页面格式（topics/）

当同类标签的文章积累到 3 篇以上时，自动生成或更新主题页面：

```markdown
---
title: 主题名称
tags: [标签]
updated: YYYY-MM-DD
related: [[相关文章1]], [[相关文章2]]
---

## 概述
基于 N 篇文章的聚合总结。

## 核心发现
- ...

## 关联文章
- [[文章1]] - 一句话摘要
- [[文章2]] - 一句话摘要
```

## 问答规则

用户提问时：
1. 先读 `wiki/index.md` 了解知识库全貌
2. 根据问题定位相关的 sources/topics/analysis 文件
3. 基于知识库内容回答，标注引用来源
4. 如果知识库中没有相关内容，如实告知

## 标签体系

AI 自动打标签，不预定义。常见维度：
- 领域：技术、股票、生活、健康、读书、工作
- 具体主题：Rust、新能源、量化交易
- 类型：教程、观点、研报、经验

## 必须使用中文

所有生成的文件内容使用中文，文件名可包含英文。
