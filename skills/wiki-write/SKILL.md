# Write — 知识创作流程

> **分层**：④ 输出类 · 从知识库出东西
> **被调**：用户
> **调用**：无（读 wiki/connections/ 的跨课题关联产出短文）

## 触发条件

- 用户说"写篇小红书"、"写个帖子"、"来一篇"、"创作"，并带有明确或隐含主题
- 用户说"写个XX话题"指定话题
- 示例："写篇SpaceX的小红书"、"来一篇美联储的"、"写个帖子讲油价"

**缺少主题时**：从 `wiki/connections/` 中找最近一条未写过草稿的关联发现，向用户提议："要不要写这篇？[提议标题]——[一句话概括]"。用户确认后开始写作，用户拒绝则提议下一条，全部拒绝则问"想写什么话题？"

🔴 **CHECKPOINT: 渠道确认**

触发词中已明确指定渠道（如"写篇小红书"）→ 跳过此步骤，使用指定渠道。

触发词中**未明确指定渠道**（如"来一篇"、"写个帖子"、"创作"）→ **必须先问用户**：

```
写哪个渠道？
1. 小红书（短文案，300-600字）
2. 技术博客（长文）
3. 随笔/杂文
```

- 用户选择后，进入对应渠道的写作流程
- 当前仅小红书有完整写作规范，其他渠道选择后告知"该渠道的写作规范尚未配置，将使用通用格式输出"
- **未获得渠道确认前不开始写作**

## 前置检查

1. **获取准确时间**（必须）：执行 `date "+%Y-%m-%d %H:%M %a"`
2. 检查环境变量 `WIKI_ROOT` 是否设置
3. 检查 `$WIKI_ROOT/drafts/rednote/` 目录是否存在（不存在则创建）
4. 读取 `$WIKI_ROOT/wiki/connections/` 下的关联分析卡片（选题素材）
5. 读取 `$WIKI_ROOT/wiki/analysis/` 下的 report.md（核心发现素材）
6. 读取 `todos/active.md` 的跟踪项（当前热点事件素材）

## 核心理念

**不是信息搬运，是连接推理。** 差异化在于：别人告诉你"CPI 4.2%，高于预期"（信息），我们告诉你"CPI 4.2% → 你买的REIT基金会跌，但JEPI反而受益"（连接）。

内容公式：
```
大家都知道的事A + 大家都知道的事B → 但没人说的结论C
```

C 从哪来？从 wiki-research 的二级效应扫描和 wiki/connections/ 的跨课题关联中来。研究阶段已经做了跨域碰撞，创作阶段只需要把碰撞结果翻译成人话。

## 执行步骤

### 1. 选题

**用户指定话题**：
1. 从 wiki/connections/ 和 wiki/analysis/ 中搜索与话题相关的二级效应和核心发现
2. 找到后进入步骤 2

**用户未指定话题**：
1. 优先从 `wiki/connections/` 中找最近的关联发现（这些已经是跨域碰撞的成果）
2. 次选从 `wiki/analysis/` 的 report.md 中找"当前主导变量"或"二级效应"节
3. 再次选从 `todos/active.md` 的跟踪项中找近期热点事件
4. 向用户提议 2-3 个选题，每个一句话概括："[标题]——[连接链]"
5. 用户选一个后进入步骤 2

**选题质量标准**：
- 必须包含至少一个"非显而易见的连接"（不是简单复述新闻）
- 必须与普通人的钱袋子/生活有关联（不能太抽象）
- 时效性：优先选近 1-2 周的事件/数据

### 2. 提取素材

从 wiki 中提取以下素材：

1. **核心连接**（必须）：从 connections/ 或 analysis/ 的二级效应节中提取 A→B→C 的因果链
2. **数据支撑**（必须）：从 report.md 和 materials/ 中提取关键数字，标注时效
3. **历史类比**（可选）：如果研究中搜索过历史类比，直接复用
4. **对冲/反面**（可选）：报告中情景分析的不同情景，可作为文章的"但如果XX"补充

⚠️ **禁止凭空编造数据**：所有数字必须来自 wiki 中已有的研究材料。如果素材不够，向用户说明"这个话题的数据还不够扎实，建议先 /wiki-research 补充"，而不是用未经搜索验证的数字。

### 3. 写作

根据目标平台选择写作风格。**默认：小红书**。

#### 小红书风格规范

**结构**（固定）：

```
[标题：口语化、有悬念、暗示"你不知道的事"]

[开头：用大家都知道的新闻/现象做钩子，1-3句话拉读者进来]

[中间：展开 A+B→C 的连接推理。用短句、用比喻、用生活化的类比。
不要堆数据，只保留最关键的1-2个数字。
用"→"或"这事的传导链是"来展示因果链。]

[结尾：点题。回到读者视角，告诉读者"所以这件事跟你有什么关系"。]

[系列说明：固定尾部，说明内容方法论]
```

**文风要求**：
- 口语化、短句为主，不用长从句
- 不用学术黑话（"传导机制"→"传递链"、"估值中枢"→"定价天花板"）
- 用第二人称"你"拉近距离
- 每段不超过 3-4 句
- 关键判断用**加粗**
- 不用 emoji（除非用户要求）
- 不用"我们来聊聊"、"今天想和大家分享"等客套开头，直接进正题
- 禁止"总结一下"、"总的来说"等总结性收尾，结尾必须是新信息或点题

**篇幅**：300-600字（小红书短内容，不是长文）

**内容红线**：
- **不做投资建议**：不说"你应该买/卖XX"，只说"XX和XX之间存在这样的关联"
- **不做恐慌贩卖**：不用"崩盘"、"灾难"、"赶紧跑"等极端措辞
- **标注不确定性**：如果因果链中某一环是假设/推测，用"可能"、"如果XX的话"
- **时效性标注**：文章中提到的事件必须带具体日期（如"6月12日"），不能写"最近"、"日前"

### 4. 输出

创建草稿文件到 `$WIKI_ROOT/drafts/rednote/`：

文件名：`YYYY-MM-DD-{slug}.md`（slug 从标题关键词生成，中英文均可）

frontmatter 格式：
```markdown
---
title: 文章标题
platform: rednote
topic: 来源课题
series: 宏观连线
status: draft
created: YYYY-MM-DD
tags: [标签1, 标签2]
sources:
  - wiki/connections/XXX.md
  - wiki/analysis/XXX/report.md
---

[正文]
```

向用户展示草稿正文（不展示 frontmatter）。

### 5. 修改与定稿

用户反馈后修改：
- "太长了" → 精简到 300 字以内
- "太干了" → 加生活化类比
- "太浅了" → 补充一层传导链
- "标题不行" → 重新起 3 个标题供选择
- "可以" → 标记 `status: published`
- "再写一篇" → 回到步骤 1

修改后直接更新同一文件，不改文件名。

## 平台扩展

当前只支持小红书（`platform: rednote`）。后续可扩展：

| 平台 | 目录 | 触发词 |
|------|------|--------|
| 小红书 | `drafts/rednote/` | "写篇小红书"（默认） |
| 技术博客 | `drafts/tech/` | "写篇技术文章" |
| 随笔/杂文 | `drafts/essay/` | "写篇随笔" |

新增平台时，在步骤 3 中增加对应平台的风格规范。

## 选题库

如果用户说"有什么可写的"或"给我推荐选题"：

1. 扫描 `wiki/connections/` 下所有卡片
2. 对每张卡片，检查 `drafts/rednote/` 下是否已有同话题草稿（避免重复）
3. 对未写过的话题，生成一个推荐标题和一句话概括
4. 按时效性排序（最近的事件优先）
5. 向用户展示 5 个推荐选题

## 反例与黑名单

1. **不要搬运新闻**：如果文章只是复述"X公司发了财报，营收Y亿"而没有跨域连接，重写
2. **不要编造数据**：所有数字必须有 wiki 素材来源，没有就先做研究再写
3. **不要做投资建议**：不说"应该买/卖"，只展示连接
4. **不要用学术腔**：不用"综上所述"、"值得注意的是"、"我们不妨来看"
5. **不要写长文**：超过 600 字就太长了，砍
6. **不要重复已有内容**：写之前检查 drafts/rednote/ 是否已有同话题草稿
7. **不要省略 sources**：frontmatter 必须标注来源课题和 connections 文件，确保可追溯
8. **不要自说自话**：每篇文章必须回答"跟读者有什么关系"，不能只讲宏观逻辑不落地
