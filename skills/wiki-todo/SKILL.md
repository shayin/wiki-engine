# Todo — 待办管理 + 闹钟

## 触发条件

- "加个待办"、"记一个待办"、"TODO" → **添加待办**
- "完成了XX"、"XX搞定了" → **完成**
- "看看待办"、"有哪些待办"、"待办列表" → **列表**
- "删掉XX"、"不要XX了" → **删除**
- "把XX改成YY"、"修改XX" → **编辑**
- "XX又要做了"、"重新打开XX" → **重新打开**
- "归档本周"、"归档待办" → **归档**
- "设个闹钟"、"加个闹钟"、"提醒我XX"（无待办上下文） → **添加闹钟**
- "有哪些闹钟"、"看看闹钟" → **列表**（显示闹钟区）

## 前置检查

1. **定位 wiki 根目录**：
   - 检查环境变量 `WIKI_ROOT` 是否设置。未设置则报错："未设置 WIKI_ROOT 环境变量，无法定位知识库。请运行 `echo $WIKI_ROOT` 检查。"
   - 验证：`$WIKI_ROOT/todos/active.md` 是否存在
2. 读取 active.md，如果内容为空或格式严重损坏（缺少 `# 待办` 标题），重建为空模板：

```markdown
# 待办
## 工作
### 长期
（暂无）
## 个人
### 长期
（暂无）
## 闹钟
（暂无）
## 跟踪项
## 已完成（本周）
### 工作
### 个人
```

## 操作

### 添加待办

1. 读取 `todos/active.md`
2. 判断分类：
   - 工作项目、团队、技术任务、会议 → **工作**
   - 个人学习、生活、投资、健康 → **个人**
   - 不确定时直接问用户："这是工作还是个人的？"
3. 判断是否长期：
   - 持续性、无明确截止日期、需要周期性跟进 → **长期**
   - 一次性任务 → **普通**
4. 追加到对应位置：
   - 普通：`## 工作` 或 `## 个人` 主列表末尾（`### 长期` 之前）
   - 长期：对应分类的 `### 长期` 下
   - 格式：`- [ ] 待办描述 \`created: YYYY-MM-DD\``

### 添加待办（带提醒）

当用户提到提醒时间时（如"下午 3 点提醒我"、"明天上午 10 点"、"每周五 9 点"），自动添加 `remind` 标记：

1. 解析用户的提醒时间，转换为标准格式：
   - "下午 3 点" / "15 点" → `remind:15:00`
   - "明天上午 10 点" → `remind:5/17 10:00`（具体日期）
   - "每周五 9 点" → `remind:Fri 09:00`
   - "每天 8 点" → `remind:08:00`
2. 追加到待办条目末尾：`- [ ] 待办描述 \`created: YYYY-MM-DD\` \`remind:HH:MM\``

提醒类型说明：
- `remind:HH:MM` — 每天该时间提醒
- `remind:DAY HH:MM` — 每周指定日提醒（Mon/Tue/Wed/Thu/Fri/Sat/Sun）
- `remind:M/D HH:MM` — 指定日期提醒一次（到期后用户确认时自动标记完成）

不提及提醒时间的待办不受影响，只出现在早晚汇总中。
   - 长期待办额外加 cadence：`- [ ] 待办描述 \`cadence: monthly\` \`created: YYYY-MM-DD\``
5. 回复："已添加：待办描述 #工作/#个人"

### 完成待办

1. 读取 `todos/active.md`
2. 在未完成列表中模糊匹配（关键词匹配，不需要完全一致）
3. **精确匹配到 1 条**：直接完成，从原位置移除，追加到「已完成（本周）」对应子分区
4. **匹配到多条**：列出匹配项让用户选择编号
5. **无匹配**：列出当前所有未完成待办让用户指定
6. 完成格式：`- [x] 待办描述 \`completed: YYYY-MM-DD\``
7. 回复："已完成：待办描述"

### 列表

1. 读取 `todos/active.md`
2. 输出未完成待办，按「工作」和「个人」分组，每组内区分普通和长期
3. 长期待办额外显示 cadence
4. 不显示已完成的项目

### 删除待办

1. 读取 `todos/active.md`
2. 模糊匹配未完成列表中的待办（同完成操作的匹配逻辑）
3. 匹配到后向用户确认："确认删除「XX」？"
4. 确认后从 active.md 中移除该条
5. 回复："已删除：待办描述"

### 编辑待办

1. 读取 `todos/active.md`
2. 模糊匹配待办
3. 更新描述文字（保留原有的 created/cadence 等元信息）
4. 如用户要求切换分类（工作↔个人），移动到对应分区
5. 回复："已更新：旧描述 → 新描述"

### 重新打开待办

1. 在「已完成（本周）」中模糊匹配
2. 将该条从已完成移回对应未完成分区，`[x]` 改回 `[ ]`，移除 `completed` 字段
3. 回复："已重新打开：待办描述"

### 归档

1. 读取 `todos/active.md`
2. 如果「已完成（本周）」两个子分区都为空，回复"本周没有已完成的待办，无需归档。" 并停止
3. 计算当前 ISO 周数，生成年周标识如 `2026-W20`
4. 计算本周日期范围（周一到周日）
5. 如果 `todos/archive/YYYY-WXX.md` 已存在，在文件名后追加 `-2`（如 `2026-W20-2.md`）
6. 创建归档文件：
   ```markdown
   # YYYY 第XX周（MM-DD ~ MM-DD）

   ## 已完成
   ### 工作
   - [x] 待办1 `completed: YYYY-MM-DD`
   ### 个人
   - [x] 待办2 `completed: YYYY-MM-DD`

   ## 未完成（顺延）
   - [ ] 待办3 `created: YYYY-MM-DD`
   ```
   顺延项 = 普通（非长期）未完成项
7. 更新 active.md：
   - 清空「已完成（本周）」两个子分区
   - 移除已顺延的普通未完成项（它们已记录在归档文件中）
   - **保留**：长期待办、本周新增但未完成的普通待办
8. 回复："已归档到 archive/YYYY-WXX.md，工作完成 X 项，个人完成 Y 项，顺延 Z 项"

## 闹钟操作

闹钟与待办的区别：待办有完成状态，闹钟到点响铃就完事，不需要完成。

### 添加闹钟

1. 读取 `todos/active.md`
2. 根据语义判断是否循环：
   - 循环关键词："每天"、"每晚"、"每个"、"每周X"、"工作日" → **循环**
   - 无循环关键词 → **一次性**
3. 追加到 `## 闹钟` 区末尾：
   - 一次性：`- ⏰ 描述 `remind:M/D HH:MM``（触发后自动删除）
   - 每天：`- ⏰ 每天 HH:MM 描述 `remind:HH:MM``
   - 每周：`- ⏰ 周X HH:MM 描述 `remind:DAY HH:MM``
4. 回复："已设闹钟：描述（一次性/每天/每周X）"

### 删除闹钟

1. 在 `## 闹钟` 区模糊匹配
2. 确认后从 active.md 中移除
3. 回复："已删除闹钟：描述"

### 列表

"看看闹钟"或"有哪些闹钟"时，输出 `## 闹钟` 区的所有条目。列表命令同时展示待办和闹钟。

## 问答

- "这周完成了什么" → 读 active.md 的「已完成（本周）」
- "上周完成了什么" → 读 `todos/archive/` 下对应周文件
- "历史完成情况" → 列出 archive/ 下所有文件及其日期范围

## 路径约定

所有操作基于 `$WIKI_ROOT`。路径为 `$WIKI_ROOT/todos/active.md` 和 `$WIKI_ROOT/todos/archive/`。
