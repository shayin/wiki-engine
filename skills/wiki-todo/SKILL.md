# Todo — 待办管理

## 触发条件

- 用户说"加个待办"、"记一个待办"、"TODO" 等 → **添加**
- 用户说"完成了XX"、"XX搞定了" → **完成**
- 用户说"看看待办"、"有哪些待办"、"待办列表" → **列表**
- 用户说"归档本周"、"归档待办" → **归档**

## 前置检查

当前目录下必须有 `todos/active.md`。如果没有，提示用户切换到 wiki 根目录。

## 操作

### 添加待办

1. 读取 `todos/active.md`
2. 判断分类：工作 or 个人
   - 涉及工作项目、团队、技术任务 → 工作
   - 个人学习、生活、投资 → 个人
   - 不确定时问用户
3. 在对应 section 下追加：
   ```
   - [ ] 待办描述 `created: YYYY-MM-DD`
   ```
4. 回复用户："已添加：待办描述 #工作/#个人"

### 完成待办

1. 读取 `todos/active.md`
2. 在未完成列表中找到匹配的待办（模糊匹配，不用完全一致）
3. 将该条从原位置移除，追加到「已完成（本周）」section：
   ```
   - [x] 待办描述 `completed: YYYY-MM-DD`
   ```
4. 回复用户："已完成：待办描述"

如果没有找到匹配项，列出当前待办让用户选择。

### 列表

1. 读取 `todos/active.md`
2. 输出未完成的待办，按分类分组
3. 不显示已完成的项目

### 归档

1. 读取 `todos/active.md`
2. 计算当前周数（ISO week），生成年周标识如 `2026-W20`
3. 计算本周日期范围（周一到周日）
4. 创建 `todos/archive/YYYY-WXX.md`：
   ```markdown
   # YYYY 第XX周（MM-DD ~ MM-DD）

   ## 已完成

   ### 工作
   - [x] 待办1 completed日期

   ### 个人
   - [x] 待办2 completed日期

   ## 未完成（顺延）
   - [ ] 待办3 created日期
   ```
5. 清空 `todos/active.md` 的「已完成（本周）」section（重置为空的工作/个人子分区）
6. 保留未完成的项目在原位
7. 回复用户："已归档本周待办到 archive/YYYY-WXX.md，工作完成 X 项，个人完成 Y 项，顺延 Z 项"

## 问答

用户问"这周完成了什么"时，读取 `todos/active.md` 的「已完成（本周）」section。
用户问"上周完成了什么"时，读取 `todos/archive/` 下对应周文件。

## 文件位置

所有操作基于当前工作目录，使用相对路径 `todos/active.md` 和 `todos/archive/`。
