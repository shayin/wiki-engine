# Review — 决策复盘流程

## 触发条件

- 用户说"复盘"、"回顾决策"、"review"
- 用户说"看看投资决策"等带领域限定的复盘请求（按 tags 字段筛选）
- cron wrapper 传入粗筛结果时自动触发

## 前置检查

1. 检查环境变量 `WIKI_ROOT` 是否设置。未设置则报错："未设置 WIKI_ROOT 环境变量，无法定位知识库。请运行 `echo $WIKI_ROOT` 检查。"
2. 检查 `$WIKI_ROOT/wiki/` 目录是否存在
3. 检查 `$WIKI_ROOT/decisions/` 目录是否存在且包含 .md 文件
4. 不满足时报错并提示检查 WIKI_ROOT 配置

## 执行模式

### 模式 A：交互模式（用户手动触发）

完整扫描 + 逐条交互复盘。

### 模式 B：精析模式（cron wrapper 传入粗筛结果）

当 prompt 中包含「粗筛已发现以下问题」时激活：
- 只读取粗筛命中的决策文件
- 不进行交互式复盘（无用户输入）
- 列出到期项并写入 log.md，通知用户手动触发

### 粗筛结果格式

由 review-check.sh 生成，每行一条：
```
decisions/xxx.md|决策标题|2026-05-01
```
字段：文件路径 | 标题 | review_date

## 执行步骤

### 1. 扫描待复盘决策

**交互模式**：读取 `decisions/` 下所有文件，通过 frontmatter 筛选 `status: open` 且 `review_date <= 今天`。

**标签筛选**：如果用户指定了领域（如"投资决策"、"工作相关"），在上述筛选基础上进一步按 `tags` 字段过滤：
- 用户说"投资决策" → 只看 tags 含"投资"的决策
- 用户说"看看我的工作决策" → 只看 tags 含"工作"或"职业"的决策
- 无匹配时回复"没有该领域的到期决策"并列出所有到期决策供参考

如果 decisions/ 下没有 .md 文件或所有决策都不是 open 状态，跳到步骤 2 的"无待复盘"分支。

**精析模式**：只读取粗筛结果中的文件路径。文件不存在则跳过该条。

### 2. 判断是否有待复盘项

**无待复盘项**：
- 回复"当前没有到期的决策需要复盘。"
- 列出所有 open 状态决策的标题和 review_date（如有），供参考
- 输出 `NOTIFY: review 完成：当前无到期决策`
- 停止执行

**有待复盘项（交互模式）**：
- 告知用户："有 N 条决策到期待复盘。"
- 进入步骤 3 逐条复盘

**有待复盘项（精析模式）**：
- 列出每条的标题、review_date、距今天数
- 写入 `wiki/log.md`：`- YYYY-MM-DD HH:mm: [auto] N 条决策到期待复盘（{标题列表}），等待手动触发 /wiki-review`
- 输出 `NOTIFY: review 完成：N 条决策到期待复盘，请手动触发复盘`

### 3. 逐条复盘（仅交互模式）

对每条待复盘决策，一次只展示一条：

1. **展示背景**（决策、理由、预期三部分，不展示完整文件）：
   ```
   📋 决策：{决策内容}
   📝 理由：{当时理由}
   🎯 预期：{预期结果}
   📅 决策时间：{date}  到期：{review_date}
   ```

2. **请用户回顾**（引导性问题）：
   - "实际发生了什么？"
   - "和预期一致吗？"
   - "有什么教训？"
   - 用户可以说"跳过"留待下次

3. **记录复盘**：
   - 在决策文件的 `## 复盘` 部分写入用户反馈，格式：
     ```
     ### YYYY-MM-DD 复盘
     - **实际结果**：{用户描述的实际发生情况}
     - **与预期对比**：{一致/偏差、偏差在哪}
     - **教训**：{用户总结的教训}
     ```
   - 更新 frontmatter：`status: reviewed`，追加 `reviewed_date: YYYY-MM-DD`

4. **追问**（可选）：
   - 是否需要继续跟进？如需要，建议新的 review_date（默认 3 个月后）
   - 如决策涉及持续变化的数据，建议创建 follow-up 跟踪项
   - 复盘中是否有共性模式？是否需要更新 `context/` 中的个人偏好？
   - 是否有关联的其他决策也值得一起复盘？

### 4. 复盘总结（仅交互模式）

全部复盘完成后：
- 汇总本次复盘条数
- 提炼共性教训（如有）
- 如发现涉及 `context/` 的偏好变化（如投资风格、风险偏好），建议用户更新
- 写入 `wiki/log.md`：`- YYYY-MM-DD HH:mm: 完成决策复盘，共 N 条。共性发现：...`
- 输出 `NOTIFY: review 完成：复盘 N 条决策`

## 用户交互规则

- 一次一条，等用户回复后再继续
- 引导但不替用户总结——复盘内容由用户说，AI 记录
- 允许"跳过"

## 错误处理

- **decisions/ 为空**：告知"还没有决策记录"
- **决策文件 frontmatter 缺少 status/review_date**：跳过该文件，告知用户
- **用户中途退出**：已复盘的正常保存，未复盘的保持 open

## 路径约定

所有路径基于 `$WIKI_ROOT`。操作前先验证 `WIKI_ROOT` 环境变量已设置。
