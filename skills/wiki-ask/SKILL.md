# Ask — 知识库问答

## 触发条件

- 用户提出问题（非指令性输入，如不是"收录"、"研究"、"加个待办"等）
- 用户说"问一下"、"帮我看看"等
- 用户提到知识库中可能有相关内容的任何问题

## 前置检查

当前目录下必须有 `wiki/` 目录。如果没有，报错并提示用户切换到 wiki 根目录。

## 执行步骤

### 1. 理解问题

分析用户问题的核心意图：
- 在问什么概念/事物/数据？
- 期望什么类型的回答（事实、观点、对比、趋势）？
- 是否涉及个人决策（需要读取 `context/`）？

### 2. 搜索知识库

**第一轮：定位候选文件**

1. 读取 `wiki/index.md`，获取知识库全貌和最近更新列表
2. 根据问题关键词，在以下目录中搜索：
   - `wiki/sources/` — 按标题、标签、关键词匹配
   - `wiki/topics/` — 按主题名称和标签匹配
   - `wiki/analysis/` — 按课题名、report 内容匹配
   - `decisions/` — 按决策标题和标签匹配

3. 搜索策略：
   - 从问题中提取 2-4 个关键词
   - 优先匹配 tags 字段，其次匹配标题和内容
   - 每个目录取相关度最高的 3-5 篇

**第二轮：精读候选文件**

4. 读取第一轮命中的文件，判断相关性：
   - 高相关：直接回答问题或包含关键数据
   - 低相关：仅作为补充背景
   - 不相关：丢弃

5. 如果高相关文件不足 2 篇，扩大搜索范围：
   - 读取候选文件的 `related` 字段，顺藤摸瓜
   - 用更宽泛的关键词重新搜索

### 3. 读取个人上下文（按需）

如果问题涉及个人决策（如"我该不该买XX"、"这个跟我的研究方向有什么关系"）：
- 读取 `context/` 下相关文件
- 将知识库内容与个人上下文结合分析

### 4. 组织回答

**有相关内容时**：

```
{直接回答问题}

来源：
- [文章A](sources/xxx.md)：{一句话说明贡献了什么信息}
- [主题B](topics/xxx.md)：{一句话说明贡献了什么信息}

{如果有个人上下文} → 补充"基于你的情况..."建议
```

**知识库无相关内容时**：

```
知识库中暂无相关内容。

{可选}：
- 我知道的一些背景信息：...（标注为通用知识，非知识库来源）
- 如果需要深入研究，可以说"研究 XX"，我会搜集资料分析。
```

**有部分相关内容时**：

```
知识库中有部分相关信息：

{基于已有内容的回答}

但以下方面知识库尚未覆盖：
- {信息缺口 1}
- {信息缺口 2}

如需补充，可以说"研究 XX"。
```

### 5. 学习反馈（按需）

如果用户追问或纠正：
- 用户提供了新信息 → 建议"要不要把这条记录下来？"
- 用户指出回答有误 → 记住修正，下次更准确
- 用户问的知识确实在库里但没找到 → 检查搜索策略，思考为什么漏了

## 搜索技巧

| 场景 | 策略 |
|------|------|
| 具体股票/公司 | 搜索标签 + analysis/ 下课题名 |
| 技术概念 | 搜索 sources/ 标签 + 标题关键词 |
| 趋势/对比 | 搜索 topics/ 聚合页面 |
| 个人决策相关 | 知识库内容 + context/ 个人上下文 |
| 时间相关（"最近"） | 优先读 index.md 最近更新区 |

## 路径约定

所有路径基于当前工作目录（相对路径）。不使用配置文件，不向上查找。当前目录必须有 `wiki/` 目录才算有效的 wiki 根目录。
