# Digest — 文章处理流程

## 触发条件

- 用户发送 URL（http/https 开头）
- 用户说"处理"、"收录"、"digest" + 附带链接或文本
- 用户发送一段长文本（非 URL，视为自己写的文章，`source: self`）

**缺少输入时**：回复"请提供文章链接或直接粘贴文章内容。"并停止执行。

**批量 URL**：用户一次发送多个 URL 时，按顺序逐个处理，每个走完整流程。每篇完成后简短回复，全部完成后汇总。

## 前置检查

1. 检查 `inbox/`、`raw/`、`wiki/` 三个目录是否存在
2. 检查 `wiki/index.md` 和 `wiki/log.md` 是否存在
3. 不满足时报错："当前目录不是有效的 wiki 根目录。请 cd 到包含 inbox/、raw/、wiki/ 的目录。"

## 执行步骤

### 1. 获取内容

**URL 模式**：
- 使用 `mcp__web-reader__webReader` 抓取，参数：`retain_images: true, return_format: markdown`
- 失败时回退 `mcp__web_reader__webReader`
- 都失败则告知用户，建议手动粘贴正文

**微信公众号特殊处理**：
- `mp.weixin.qq.com` 链接大概率抓取失败或内容不全
- 如果是微信链接，优先提示用户："微信文章可能抓取不完整，如果内容不对请直接粘贴正文。"
- 抓取后检查内容长度，<500 字则建议用户手动粘贴

**长文本模式**：
- 直接使用用户文本，标记 `source: self`

**超长内容处理**（>10000 字）：
- 知识卡片的「关键要点」提炼 5-8 条（而非 3-5 条）
- 原文照常归档到 raw/，不截断

**提取用户备注**：
- 用户消息中除 URL 外的额外文字（如"这篇很重要"、"关于XX的"）提取为备注
- 备注用于辅助打标签和生成摘要

### 2. 疑似重复检查

检查 `wiki/sources/` 是否已有：
- 相同 URL（匹配 frontmatter `origin` 字段）
- 高度相似标题（模糊匹配）

如命中，提示用户："这篇文章可能已收录为「{已有标题}」，确认是否重复？"确认重复则停止，用户说不是重复则继续。

### 3. 存入 inbox/

保存到 `inbox/YYYY-MM-DD-简短标识.md`：
- 简短标识：标题前 2-4 个关键词，短横线连接，全小写，去特殊字符
- 示例：`2026-05-14-rust-async-runtime.md`

```markdown
---
url: 原始链接（如有）
fetched: YYYY-MM-DD HH:mm
status: pending
---

原始文章标题

原始正文内容...
```

### 4. 生成知识卡片

先读 `wiki/index.md` 了解知识库全貌，再按 CLAUDE.md 中「知识卡片格式」生成：

**source: article（外部文章）**：
- frontmatter：title、date、tags、source、origin、related
- 一句话摘要（50 字内）
- 关键要点（3-5 条）
- 原文链接

**source: self（用户写的）**：
- 保留原文不重写
- 只做分类、打标签、建关联
- 摘要从原文首段提取或让用户补充

**标签规则**：
- 两层：领域（技术/股票/生活/工作/读书...）+ 具体主题（Rust/量化交易...）
- 优先复用 `wiki/sources/` 下已有标签
- 每篇 2-5 个标签
- related 通过搜索 wiki/ 找相关页面，用 `[[]]` 双链

### 5. 写入 wiki/sources/

保存到 `wiki/sources/YYYY-MM-DD-简短标识.md`

### 6. 归档原文

创建 `raw/YYYY-MM/`（如不存在），将 inbox/ 中的原文移过去。

### 7. 更新 wiki/index.md

在「最近更新」追加：
```
- YYYY-MM-DD: [文章标题](sources/YYYY-MM-DD-简短标识.md) #标签1 #标签2
```
更新统计数字。

### 8. 检查 topic 更新

检查本次文章的标签：
- 如果 `wiki/topics/` 下已有对应主题页面 → 追加到关联文章列表
- 如果某标签下文章累计 >=3 篇且无 topic → 问用户："XX 主题已有 3 篇文章，是否创建主题页面？"

### 9. 检查跟进项关联

扫描 `wiki/analysis/*/follow-ups/` 下 `status: active` 的跟进项：
- 文章标签与跟进项 topic 匹配，或文章内容包含跟进项跟踪的指标/数据
- 匹配到时提醒用户："这篇文章跟你跟进的「{跟进项名}」相关，要更新跟踪记录吗？"
- 用户确认后追加到跟进项的 `## 跟踪记录`：`- **YYYY-MM-DD**: 更新——xxx，来源 [[sources/本文]]`

### 10. 更新 wiki/log.md

追加：`- YYYY-MM-DD HH:mm: 处理了「文章标题」→ sources/YYYY-MM-DD-简短标识.md`

### 11. 回复用户

```
已收录「文章标题」

一句话摘要。

#标签1 #标签2
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 缺少输入 | 提示发送链接或粘贴正文 |
| 抓取失败 | 告知用户，建议手动粘贴 |
| 内容 <100 字 | 告知用户，确认是否仍要收录 |
| 文件名冲突 | 追加序号（`-2`） |
| 疑似重复 | 提示用户确认 |

## 路径约定

所有路径基于当前工作目录（相对路径）。不使用配置文件，不向上查找。当前目录必须有 `inbox/`、`raw/`、`wiki/` 三个目录才算有效的 wiki 根目录。
