# Digest — 文章处理流程

## 触发条件

- 用户发送了一个 URL（http/https 开头的链接）
- 用户说"处理"、"收录"、"digest"等相关指令
- 用户发送一段长文本（非 URL，视为自己写的文章）

**缺少输入时：** 用户只说"处理/收录"但没有附带链接或文本，回复"请提供文章链接或直接粘贴文章内容。" 并停止执行。

## 前置检查

执行前必须检查当前工作目录是否符合 wiki 规范：

1. 检查当前目录下是否存在 `inbox/`、`raw/`、`wiki/` 三个目录
2. 检查 `wiki/index.md` 和 `wiki/log.md` 是否存在
3. 如果不满足，报错并提示用户切换到 wiki 根目录：
   ```
   当前目录不是有效的 wiki 根目录。请 cd 到包含 inbox/、raw/、wiki/ 的目录。
   ```

所有文件操作均基于当前工作目录，使用相对路径。

## 执行步骤

### 1. 获取内容

**如果是 URL：**
- 使用 web fetch 工具抓取文章全文
- 提取标题、正文、来源
- 如果抓取失败，告知用户并建议手动粘贴正文
- 如果 URL 来自微信公众号（mp.weixin.qq.com），抓取很可能失败，优先提示用户粘贴正文或截图

**如果是长文本：**
- 视为用户自己写的文章（`source: self`）
- 直接使用用户提供的文本

**提取用户备注：**
- 如果用户消息中除 URL 外还有额外文字（如"这篇很重要"、"关于XX的文章"），提取为用户备注
- 用户备注用于辅助打标签和生成摘要，可写入知识卡片的 `note` 字段

### 2. 存入 inbox/

将原始内容保存到 inbox/，文件名格式：`YYYY-MM-DD-简短标识.md`
- 简短标识规则：取文章标题的前 2-4 个关键词，用 `-` 连接，全小写，去掉特殊字符
- 示例：`2026-05-14-rust-async-runtime.md`、`2026-05-14-量化交易策略.md`

```markdown
---
url: 原始链接（如有）
fetched: YYYY-MM-DD HH:mm
status: pending
---

原始文章标题

原始正文内容...
```

### 3. 生成知识卡片

先读取 `wiki/index.md` 了解知识库全貌，再按 **wiki-engine/CLAUDE.md 中「知识卡片格式」** 生成卡片。不要自行简化格式——必须包含完整的 frontmatter（title/date/tags/source/origin/related）、一句话摘要、关键要点、原文链接。

**对于 source: article（外部文章）：** 按 wiki-engine/CLAUDE.md 的知识卡片模板生成，包含一句话摘要和关键要点。

**对于 source: self（用户自己写的）：** 保留用户原文不重写，只做分类打标签和建立关联。

标签规则：
- 自动打标签，基于文章内容判断
- 标签分两层：领域（技术/股票/生活/工作/读书...）+ 具体主题（Rust/量化交易/新能源...）
- 优先复用 wiki/sources/ 下已有文章的标签，保持一致性
- 每篇文章 2-5 个标签
- related 通过搜索 wiki/ 下已有内容找出相关页面，用 `[[]]` 双链语法链接

### 4. 写入 wiki/sources/

将知识卡片保存到 `wiki/sources/YYYY-MM-DD-简短标识.md`

### 5. 归档原文

创建 `raw/YYYY-MM/` 目录（如不存在），将 inbox/ 中的原文移过去。

### 6. 更新 wiki/index.md

在 index.md 的「最近更新」部分追加一条记录：
```
- YYYY-MM-DD: [文章标题](sources/YYYY-MM-DD-简短标识.md) #标签1 #标签2
```

更新统计数字。

### 7. 检查是否需要更新 topic

检查本次文章的标签，如果 wiki/topics/ 下已有对应主题页面，追加到该主题的关联文章列表中。

如果某个标签下的文章达到 3 篇且还没有 topic 页面，先向用户确认："_XX_ 主题已有 3 篇文章，是否创建主题页面？" 用户确认后再创建。

### 8. 更新 wiki/log.md

追加处理日志：
```
- YYYY-MM-DD HH:mm: 处理了「文章标题」→ sources/YYYY-MM-DD-简短标识.md
```

### 9. 回复用户

简洁回复，格式：
```
已收录「文章标题」

一句话摘要。

#标签1 #标签2
```

## 错误处理

- **缺少输入：** 用户只说处理/收录但没提供链接或文本，提示用户发送链接或粘贴正文
- **抓取失败：** 告知用户，建议手动粘贴正文
- **微信公众号：** mp.weixin.qq.com 链接大概率抓取失败，优先提示用户
- **内容为空或太短（<100字）：** 告知用户，确认是否仍要收录
- **文件名冲突：** 在文件名后追加序号
- **疑似重复：** 检查 wiki/sources/ 是否已有相同 URL 或高度相似标题，如有则提示用户确认是否重复收录

## 路径约定

所有路径基于当前工作目录（相对路径）。不使用配置文件，不向上查找。当前目录必须有 `inbox/`、`raw/`、`wiki/` 三个目录才算有效的 wiki 根目录。
