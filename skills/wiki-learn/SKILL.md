# Learn — 学习计划

## 触发条件

- "制定学习计划"、"学习 XX"、"过一遍 XX"、"看看 XX 相关的内容"、"帮我梳理 XX 领域的知识"、"读一下 XX" → **创建学习计划**
- "继续学习 XX"、"下一步"、"继续看" → **继续学习**
- "学完了"、"看完了"、"读完了" → **标记完成**
- "看看学习进度"、"学习情况"、"看到哪了" → **查看进度**
- "学习笔记：XXX"、"阅读笔记：XXX" → **记录笔记**

## 前置检查

1. **获取准确时间**（必须）：执行 `date "+%Y-%m-%d %H:%M %a"` 获取当前时间
2. 检查 `WIKI_ROOT` 环境变量是否设置
3. 检查 `$WIKI_ROOT/wiki/learning/` 目录是否存在（不存在则创建）

## 操作

### 创建学习计划

1. 提取用户想学习的主题关键词
2. 扫描知识库相关内容（用 Grep 搜索 frontmatter tags + Glob 按文件名匹配）：
   - `wiki/sources/*.md` — Grep `tags:` 行含关键词 → 提取标题和摘要
   - `wiki/topics/*.md` — 文件名或 `title:` 字段含关键词 → 提取主题概要
   - `wiki/analysis/*/report.md` — 文件名或 `topic:` 字段含关键词 → 提取核心结论
   - `wiki/connections/*.md` — Grep 关联描述中含关键词 → 提取关联类型
   - `wiki/decisions/*.md` — Grep 决策内容中含关键词 → 提取决策摘要
3. 按学习难度和依赖关系排序，组织成四阶段学习路径
4. **展示计划概要让用户确认**（关键检查点）：
   - 输出：学习目标 + 四阶段各几项 + 总条目数
   - 用户确认 → 创建文件；用户调整 → 修改后重新确认
5. 创建 `wiki/learning/YYYY-MM-DD-主题.md`

**排序原则**：
- 第一阶段（基础）：概念介绍、背景知识的 sources
- 第二阶段（深入）：topics 跨文章总结、analysis 研究报告
- 第三阶段（关联）：connections 跨课题关联、其他相关课题
- 第四阶段（内化）：从核心发现提炼思考题

**内容过多时筛选**（每个阶段最多 5 项，总计不超过 20 项）：
- 同一来源多篇 → 优先选与主题直接相关的，标注"选读"其余
- analysis 报告 → 只链接 report.md 的核心结论章节，不要求全文
- connections → 只选与本主题直接相关的关联类型

**每个条目格式**：`- [ ] [标题](相对路径) — 一句话说明为什么读这篇/这步重要`

**如果知识库内容不足**：
- 诚实告知："知识库中关于 XX 的内容较少（只有 X 篇），建议先研究补充"
- 仍然创建计划，标注哪些环节需要外部补充
- 可建议用 `/wiki-research` 补充研究

### 继续学习

1. 在 `$WIKI_ROOT/wiki/learning/` 下用 Grep 搜索 `topic:` 字段匹配用户提到的主题，或 Glob 匹配文件名含关键词
2. **如果匹配到多个计划**：列出供用户选择，不要自己猜
3. **如果无匹配**：列出所有 learning/ 下的计划供选择，或提示"还没有学习计划，要不要先创建？"
4. 找到第一个未完成的 `[ ]` 项
5. 向用户展示：
   - 这一步要学什么（简要说明，1-2 句）
   - 为什么重要（这步在学习路径中的位置和作用）
   - 链接（让用户可以直接去读）
6. 用户读完说"学完了"→ 标记 `[x]`，展示下一步
7. 全部完成 → 将 status 改为 `completed`，输出学习总结

### 标记完成

1. 在 `$WIKI_ROOT/wiki/learning/` 下定位活跃计划（同"继续学习"的定位方式）
2. 在学习计划中模糊匹配当前步骤
3. 将 `[ ]` 改为 `[x]`
4. 自动展示下一步（如果有）

### 查看进度

1. 扫描 `$WIKI_ROOT/wiki/learning/*.md`，找到 `status: in_progress` 的计划
2. **如果只有一个活跃计划**：直接显示进度
3. **如果有多个**：列出供用户选择
4. 统计完成/总数，输出进度条
5. 列出已完成和待完成的条目

### 记录笔记

1. 用户说"学习笔记：XXX"时，将笔记追加到学习计划的 `## 学习笔记` 节
2. 格式：`### YYYY-MM-DD\n用户记录的笔记内容`

## 学习计划文件格式

```markdown
---
topic: 学习主题
created: YYYY-MM-DD
status: in_progress
---

## 学习目标
- 问题1：学完能回答什么
- 问题2：...

## 知识资产统计
知识库中关于本主题的：X 篇文章、Y 个主题、Z 份研究报告、W 条跨课题关联

## 学习路径

### 第一阶段：基础认知（X 项）
- [ ] [文章标题](../sources/xxx.md) — 为什么先读这篇
- [ ] [文章标题](../sources/xxx.md) — 一句话

### 第二阶段：深入理解（X 项）
- [ ] [主题页面](../topics/xxx.md) — 跨文章总结
- [ ] [研究报告](../analysis/xxx/report.md) — 核心结论

### 第三阶段：跨课题关联（X 项）
- [ ] [关联卡片](../connections/xxx.md) — 交叉洞察
- [ ] [关联课题](../analysis/yyy/report.md) — 选读相关章节

### 第四阶段：内化验证（X 项）
- [ ] 思考题 1：基于学到的内容回答...
- [ ] 思考题 2：...
- [ ] 实践建议：用这些知识做XX

## 学习笔记
（学习过程中按日期追加）
```

## 问答

- "XX 学到什么了" → 读学习计划的已完成项，输出学习摘要
- "关于 XX 的学习计划有哪些" → 列出 wiki/learning/ 下所有计划

## 失败模式编码

| 触发条件 | 一线修复 | 仍失败兜底 |
|---------|---------|-----------|
| 主题无匹配内容 | 告知用户知识库覆盖不足 | 建议用 `/wiki-research` 补充，同时创建空计划框架 |
| 学习计划不存在 | 列出已有计划供选择 | 提示"还没有学习计划，要不要先创建？" |
| 已完成所有步骤 | 标记 `status: completed`，输出学习总结 | 建议将学习笔记中值得沉淀的内容入库到 sources/ |
| learning/ 目录不存在 | 自动创建 `mkdir -p` | 创建失败则告知用户手动创建 |
| 匹配到多个计划 | 列出供用户选择，不自动猜测 | 用户输入无效选择时重新列出 |
| 知识库内容严重不足（<3 篇） | 仍然创建计划，标注"内容稀疏" | 建议先 `/wiki-research` 深入研究后再学习 |

## 反例与黑名单

1. **禁止生成外部内容** — 学习计划只能引用知识库中已有的 sources/topics/analysis/connections，不允许编造不存在的文件。
2. **禁止跳过用户确认直接创建** — 步骤 4 的 🔴 CHECKPOINT（展示计划概要）是强制的，不可跳过。
3. **禁止自动搜索补充** — 知识库内容不足时应建议用户补充，不能自行搜索外部资料填充学习计划。
4. **禁止修改知识库内容** — wiki-learn 只读取和创建学习计划文件，不修改 sources/topics/analysis 中的内容。
5. **禁止强行塞满四阶段** — 如果某个阶段确实无内容可放，留空该阶段并标注"知识库暂缺"，不要凑数。

## 路径约定

所有路径基于 `$WIKI_ROOT`。学习计划保存在 `$WIKI_ROOT/wiki/learning/`。
