# Learn — 学习计划

## 触发条件

- "制定学习计划"、"学习 XX"、"帮我梳理 XX 领域的知识" → **创建学习计划**
- "继续学习 XX"、"下一步" → **继续学习**
- "学完了"、"看完了" → **标记完成**
- "看看学习进度"、"学习情况" → **查看进度**
- "学习笔记：XXX" → **记录笔记**

## 前置检查

1. **获取准确时间**（必须）：执行 `date "+%Y-%m-%d %H:%M %a"` 获取当前时间
2. 检查 `WIKI_ROOT` 环境变量是否设置
3. 检查 `$WIKI_ROOT/wiki/learning/` 目录是否存在（不存在则创建）

## 操作

### 创建学习计划

1. 提取用户想学习的主题关键词
2. 扫描知识库相关内容：
   - `wiki/sources/` — 按标签和标题匹配相关文章
   - `wiki/topics/` — 匹配相关主题页面
   - `wiki/analysis/` — 匹配相关研究报告
   - `wiki/connections/` — 匹配跨课题关联
   - `wiki/decisions/` — 匹配相关决策记录
3. 按学习难度和依赖关系排序，组织成四阶段学习路径
4. 创建 `wiki/learning/YYYY-MM-DD-主题.md`

**排序原则**：
- 第一阶段（基础）：概念介绍、背景知识的 sources
- 第二阶段（深入）：topics 跨文章总结、analysis 研究报告
- 第三阶段（关联）：connections 跨课题关联、其他相关课题
- 第四阶段（内化）：从核心发现提炼思考题

**每个条目格式**：`- [ ] [标题](相对路径) — 一句话说明为什么读这篇/这步重要`

**如果知识库内容不足**：
- 诚实告知："知识库中关于 XX 的内容较少（只有 X 篇），建议先研究补充"
- 仍然创建计划，标注哪些环节需要外部补充
- 可建议用 `/wiki-research` 补充研究

### 继续学习

1. 读取对应学习计划文件
2. 找到第一个未完成的 `[ ]` 项
3. 向用户展示：
   - 这一步要学什么（简要说明，1-2 句）
   - 为什么重要（这步在学习路径中的位置和作用）
   - 链接（让用户可以直接去读）
4. 用户读完说"学完了"→ 标记 `[x]`，展示下一步
5. 全部完成 → 将 status 改为 `completed`，输出学习总结

### 标记完成

1. 在学习计划中模糊匹配当前步骤
2. 将 `[ ]` 改为 `[x]`
3. 自动展示下一步（如果有）

### 查看进度

1. 读取学习计划文件
2. 统计完成/总数，输出进度条
3. 列出已完成和待完成的条目

### 记录笔记

1. 用户说"学习笔记：XXX"时，将笔记追加到学习计划的 `## 学习笔记` 节
2. 格式：`### YYYY-MM-DD\n用户记录的笔记内容`

## 学习计划文件格式

```markdown
---
topic: 学习主题
created: YYYY-MM-DD
status: in_progress
---

## 学习目标
- 问题1：学完能回答什么
- 问题2：...

## 知识资产统计
知识库中关于本主题的：X 篇文章、Y 个主题、Z 份研究报告、W 条跨课题关联

## 学习路径

### 第一阶段：基础认知（X 项）
- [ ] [文章标题](../sources/xxx.md) — 为什么先读这篇
- [ ] [文章标题](../sources/xxx.md) — 一句话

### 第二阶段：深入理解（X 项）
- [ ] [主题页面](../topics/xxx.md) — 跨文章总结
- [ ] [研究报告](../analysis/xxx/report.md) — 核心结论

### 第三阶段：跨课题关联（X 项）
- [ ] [关联卡片](../connections/xxx.md) — 交叉洞察
- [ ] [关联课题](../analysis/yyy/report.md) — 选读相关章节

### 第四阶段：内化验证（X 项）
- [ ] 思考题 1：基于学到的内容回答...
- [ ] 思考题 2：...
- [ ] 实践建议：用这些知识做XX

## 学习笔记
（学习过程中按日期追加）
```

## 问答

- "XX 学到什么了" → 读学习计划的已完成项，输出学习摘要
- "关于 XX 的学习计划有哪些" → 列出 wiki/learning/ 下所有计划

## 错误处理

- **主题无匹配内容**：告知用户知识库中缺少相关内容，建议先 `/wiki-research` 或 `/wiki-digest` 补充
- **学习计划不存在**：列出已有计划供选择
- **已完成所有步骤**：输出祝贺 + 学习总结，标记 completed

## 路径约定

所有路径基于 `$WIKI_ROOT`。学习计划保存在 `$WIKI_ROOT/wiki/learning/`。
