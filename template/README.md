# AI Wiki

个人知识管理与研究工作区。用户只做三件事：**存、问、研究**。中间的整理、调研、分析，由 AI 完成。

## 目录结构

```
ai-wiki/
├── inbox/          # 待处理文章入口（微信转发、浏览器插件、手动放入）
├── raw/            # 原文归档（按月 YYYY-MM/，不可变）
├── context/        # 个人上下文（用户维护，AI 只读）
├── decisions/      # 决策日志（AI 写入，定期复盘）
├── todos/          # 待办管理
├── .cron/          # 定时任务（配置 + 日志 + 脚本）
├── wiki/           # AI 维护的知识库
│   ├── index.md    #   总索引（AI 入口点）
│   ├── log.md      #   处理日志
│   ├── sources/    #   知识卡片（每篇文章一张）
│   ├── topics/     #   跨文章聚合主题（同类 ≥3 篇自动生成）
│   └── analysis/   #   课题研究工作区
└── wiki-engine/    # 知识库引擎（可复用模板）
```

## 核心流程

| 用户动作 | 触发 | AI 行为 |
|---------|------|--------|
| 存文章 | `/wiki-digest` | 生成知识卡片 → 归档原文 → 更新索引 |
| 提问题 | `/wiki-ask` | 搜索知识库 → 基于已有内容回答 |
| 做研究 | `/wiki-research` | 搜集资料 → 分析 → 输出报告 → 跟踪 |
| 管待办 | `/wiki-todo` | 添加/完成/删除/归档 |

## 维护规则

- `context/` 由用户维护，AI 只读
- `wiki/` 由 AI 维护，用户可浏览
- `inbox/` 由用户放入，AI 处理后移走
- `decisions/` 由 AI 写入，用户回看复盘
