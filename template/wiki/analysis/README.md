# 课题研究（analysis/）

每个课题一个子目录，存放从开题到报告的完整研究过程。

## 目录结构

```
{课题名}/
├── plan.md       # 研究计划（问题、已知/未知、搜索方向）
├── materials/    # 搜集的原始资料（网页抓取等）
├── notes.md      # 研究笔记（关键发现、冲突标注、信息缺口）
├── report.md     # 最终报告（结论、来源、未解决问题）
└── follow-ups/   # 持续跟踪项
```

## 触发

用户说"研究 XX"时，由 `/wiki-research` skill 触发。

## 产出分类

- **知识型**（方法论、原理）：关键发现入库到 `sources/`，更新 `topics/`
- **决策型**（时效分析、个人建议）：保留在 `analysis/`，同时在 `decisions/` 创建记录

## 状态

- `active`：研究进行中
- `completed`：研究已完成
